# Supabase Backend

This directory contains the Supabase configuration, migrations, and Edge Functions
for the ModuLearn rebuild on Supabase + Vercel.

## Layout

```
supabase/
├── migrations/         SQL migrations applied in timestamp order
├── functions/          Edge Functions (Deno/TypeScript) — BKT, admin, simulations
├── seed.sql            Seed data run after migrations on `supabase db reset`
└── config.toml         Generated by `supabase init` (not yet committed)
```

## Local development

Once the Supabase CLI is installed and you've run `supabase init` + `supabase login`:

```bash
supabase start              # spin up local Postgres + Studio at :54323
supabase db reset           # re-apply all migrations + seed
supabase functions serve    # serve Edge Functions locally
```

## Migration naming

`YYYYMMDDHHMMSS_short_description.sql`. The Supabase CLI applies them in lexicographic order.
